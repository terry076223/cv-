# 個人履歷展示暨後台管理系統  
## 建議書／需求規格書／流程說明文件

---

## 一、文件目的（Purpose）

本文件目的在於說明「個人履歷展示暨後台管理系統」之系統建議方案、功能需求規格與操作流程設計，作為系統建置、展示與後續擴充之依據。

本系統以「**個人履歷展示**」為核心，並搭配「**僅限本人使用之後台管理介面**」，可即時維護學經歷、課程、證照、獎狀與作品內容，達成履歷資料結構化、可維護化與系統化之目標。

---

## 二、系統建置背景與動機

### 2.1 建置背景

隨著資訊工程、數據分析與全端技術之整合趨勢，傳統靜態履歷文件已難以完整呈現個人技術能力與系統設計思維。因此，透過建置一套具備前後台概念之履歷網站，可有效展示以下能力：

- 系統分析與需求拆解能力
- 前後台架構與資料流設計概念
- CRUD 資料管理實作能力
- 使用者權限與操作流程設計能力

### 2.2 建置目標

- 建立具科技感與專業形象之個人履歷展示網站
- 提供可自行維護之後台管理介面
- 將履歷、課程與成果資料結構化管理
- 作為求職、教學與系統展示之整合型作品

---

## 三、系統整體架構說明

### 3.1 系統架構概念

本系統採用「**前台展示 + 後台管理**」之架構設計：

- 前台（Frontend）
  - 提供訪客瀏覽個人履歷、學經歷、作品與課程成果
- 後台（Admin）
  - 僅限本人登入
  - 提供資料新增、編輯、刪除之管理功能

### 3.2 系統角色定義

| 角色 | 說明 |
|---|---|
| 一般訪客 | 僅能瀏覽前台履歷與展示內容 |
| 系統管理者（本人） | 可登入後台，維護所有資料 |

### 3.3 視覺與互動設計風格說明

本系統整體視覺設計以「**科技感與專業感並重**」為核心，採用深色系介面搭配高對比重點色，營造現代化資訊系統之視覺印象。

設計原則包含：

- 採用深色背景搭配霓虹色系作為重點強調（如按鈕、標題、互動元件）
- 使用卡片式（Card-based）版面配置，呈現資料模組化與結構化概念
- 透過滑動、淡入、浮現等 CSS 動畫效果，提升操作時之即時回饋感
- 前台偏向展示與品牌形象，後台偏向工具導向與管理介面風格

此設計除提升整體專業度外，亦有助於突顯系統工程、數據分析與資訊整合之技術形象。

本系統之色彩設計參考 UI/UX 教學中之配色原則，並搭配色彩組合工具進行規劃。
整體配色以深色背景作為基底，搭配藍色與紫色系作為主要識別色，
並以亮色作為互動與重點提示，兼顧科技感、可讀性與視覺層次。

色彩設計遵循以下原則：
- 以深色作為背景，強化內容聚焦與科技氛圍
- 以高對比色作為按鈕與互動元素，提升使用者操作辨識度
- 前台配色偏向品牌展示，後台配色偏向管理與工具導向

### 色彩系統（調色盤與使用建議）
- Base / 背景：`#0F1629`（深藍黑），前台與後台共同基底。
- Surface / 區塊：`#141B2E`、浮層可用 `#1B2338`；邊框/分隔線建議 `#1F2940`。
- Primary（藍）：`#4DA3FF` 主色；Hover `#5AB2FF`；Active `#3E8FE0`。
- Secondary（紫）：`#9C6BFF` 次識別；Hover `#A77CFF`；Active `#8557E0`。
- Accent / 強調：`#6CF6FF`（高亮提示）與 `#FFD166`（警示/次要高亮）。
- 成功/警告/錯誤：Success `#4ADE80`，Warning `#FBBF24`，Error `#F87171`。
- 文字：主文字 `#E8EDF7`，次文字 `#B8C3D9`，弱化說明 `#8A95AC`；反白文字 `#0B1220`（用於亮色按鈕）。
- 對比：主要按鈕建議文字使用 `#0B1220`，背景採 Primary 或 Accent，可確保 WCAG AA 對比；表單邊框至少使用 `#2A3550` 以上亮度。

使用指引：前台以 Primary + Accent 提升品牌感；後台以 Secondary 降低干擾並聚焦工具性，警示類互動統一採 Warning / Error 色。

↓

↓

## 七、技術實作說明

### 7.1 使用技術

- HTML5（Semantic Tags）
- CSS3（Flexbox / Grid / RWD）
- JavaScript（ES6+）
- localStorage（前端資料模擬）

### 7.2 架構設計說明

- 前台與後台頁面分離
- 管理邏輯集中於 admin.js
- 顯示邏輯集中於 main.js
- 未使用任何前端框架，展現基礎功力
### 7.3 資料儲存與 Prototype 設計說明

本系統目前以瀏覽器端之 localStorage 作為資料儲存機制，主要目的為：

- 作為前端系統 Prototype 與功能展示用途
- 模擬後端資料存取流程
- 降低系統建置複雜度，聚焦於架構與流程設計

未來系統可依需求，無須大幅修改前端架構，即可銜接後端 API 與正式資料庫系統。

## 九、結論

本系統不僅為一履歷展示網站，更是一套具備完整系統分析與設計概念之作品展示平台。  
透過前台展示與後台管理之分離設計，搭配資料結構化管理與清楚之操作流程規劃，能有效展現系統分析、全端技術與實務經驗。

整體視覺與互動設計以科技感為核心，兼顧專業形象與操作直覺性，使系統同時具備展示價值與實作深度。

本系統設計亦可作為日後擴充為正式履歷平台、個人品牌網站或後端整合系統之基礎架構。
